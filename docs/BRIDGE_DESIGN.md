# bridge.py — Design

How `gmux-system` will serve the **gmux-phone** Android app (and any other
remote client). This is the implementation plan for the missing
`backend/bridge.py` referenced by `gmux-phone/docs/BACKEND_CONTRACT.md`.

**Status:** spec only — not implemented. Planned for v3.8.
**Phone contract version:** 1 (locked).
**Reads also:** `~/projects/gmux-phone/docs/BACKEND_CONTRACT.md` (frozen client spec).

---

## What the bridge is

A single Python service that:
1. Listens on `ws://0.0.0.0:8767` for phone WebSocket connections
2. Serves HTTP fallback on `http://0.0.0.0:8768`
3. Subscribes to `monitor.py`'s SSE stream at `:8769/api/stream`
4. Translates between the phone's protocol and the underlying tmux + opencode
5. Validates a bearer token on every connection / request

It is the **only** thing the phone talks to. The phone does NOT talk to
`monitor.py` directly because:
- Phone schema and monitor schema differ slightly
- The phone needs `spawn_agent` / `send_text` / `permission_response`
  commands that monitor.py doesn't expose
- The phone needs the token-auth layer that's pointless on localhost

## Token system

### Storage
`~/.config/gmux/pair-tokens.json`:
```json
{
  "tokens": [
    {
      "id": "tok_01H...",
      "token": "<128 hex chars>",
      "name": "fivelidz pixel 7",
      "created_at": 1736000000,
      "last_used_at": 1736003000,
      "transport": "tailscale",
      "scope": ["status", "send", "spawn"]
    }
  ]
}
```

Generated by `gmux pair`:
```bash
$ gmux pair
🔗 Pairing token generated.
Name (optional): fivelidz pixel 7
QR shown below. Scan with gmux-phone:

  [QR code rendered with `qrencode` or pure-python qrcode]

Or share this URL: gmux://pair?host=100.73.134.20&port=8767&token=...

Token saved. Revoke with: gmux pair revoke tok_01H...
```

### Verification
Every WS connection's first message must be:
```json
{ "type": "auth", "token": "..." }
```
Bridge:
1. Loads tokens from disk (cached, reloaded on file mtime change)
2. Constant-time compares against incoming `token`
3. On match: send `{ "type":"welcome", "sessions":[...], "host_name":"..." }`
4. On mismatch: close WS with code 4401 (custom "unauthorized")
5. After 5 bad tokens in 60s from one IP: rate-limit that IP for 5 min

Every HTTP request must have:
```
Authorization: Bearer <token>
```

3. Phone connects to the relay with same token; relay forwards
4. Lets phone reach host across NAT without Tailscale

## Phone schema translation

The phone's `Agent` is a subset of monitor.py's `PaneInfo`. The bridge's
`adapter.py` does field renaming only — no data fetching.

### Monitor PaneInfo → Phone Agent

```python
def pane_to_agent(p: dict) -> dict:
    return {
        "pane_id":      p["pane_id"],
        "window":       p["window_index"],
        "name":         p["window_name"],
        "agent_type":   _detect_agent_type(p),  # see below
        "cwd":          p.get("cwd", ""),
        "state":        p["state"],   # already matches
        "last_line":    p.get("last_line", ""),
        "output_tail":  _last_n_lines(p["pane_id"], n=40),  # from tmux capture-pane
        "todos":        [{"text": t, "done": False} for t in p.get("todos", [])],
        "tokens_used":  p.get("token_in", 0) + p.get("token_out", 0),
        "uptime_s":     p.get("uptime_s", 0),
        "last_activity_s": p.get("last_activity_s", 0),
        "pending_permission": _pending_perm(p) if p["state"] == "permission" else None,
    }

def _detect_agent_type(p: dict) -> str:
    cmd = p.get("foreground_cmd", "")
    if "bun" in cmd or "opencode" in cmd: return "opencode"
    if "claude" in cmd:                   return "claude"
    if "aider"  in cmd:                   return "aider"
    if "qwen"   in cmd:                   return "qwen"
    return "shell"
```

### Monitor session view → Phone Session

monitor doesn't currently aggregate by session. The bridge does:

```python
def build_sessions(pane_state: dict) -> list[dict]:
    by_session: dict[str, list] = {}
    for pid, p in pane_state.items():
        by_session.setdefault(p["session_name"], []).append(p)
    return [
        {
            "id":   sess_name,
            "name": sess_name,
            "cwd":  panes[0].get("cwd", ""),
            "active_agent_idx": 0,
            "agents": [pane_to_agent(p) for p in sorted(panes, key=lambda x: x["window_index"])],
        }
        for sess_name, panes in by_session.items()
    ]
```

`active_agent_idx` is which pane is currently the "primary" — the bridge
remembers this per WS connection (the phone sets it via `change_agent`).

## Command handling

For each phone-sent command type the bridge calls a `tmux_ops.py` helper:

```python
# tmux_ops.py
def spawn_agent(session_id, name, agent_type, cwd, command=None):
    # If session_id doesn't exist, create it first
    subprocess.run(["tmux", "new-window", "-t", f"{session_id}:", "-n", name, "-c", cwd])
    subprocess.run(["tmux", "set-window-option", "-t", f"{session_id}:{name}", "automatic-rename", "off"])
    cmd = command or _default_command_for(agent_type, cwd)
    subprocess.run(["tmux", "send-keys", "-t", f"{session_id}:{name}", cmd, "Enter"])
    return _resolve_pane_id(session_id, name)

def send_text(pane_id, text):
    subprocess.run(["tmux", "send-keys", "-t", pane_id, text, "Enter"])

def send_key(pane_id, key):
    # Map "C-c" → Ctrl+C, "Enter" → Enter, "Escape" → Escape
    subprocess.run(["tmux", "send-keys", "-t", pane_id, key])

def permission_response(pane_id, resp):
    # Find opencode session for this pane, POST /permission/allow|deny
    sess_info = _opencode_session_for(pane_id)
    if sess_info:
        endpoint = "/permission/allow" if resp != "reject" else "/permission/deny"
        requests.post(f"http://127.0.0.1:{sess_info.port}/session/{sess_info.id}{endpoint}",
                      params={"directory": sess_info.cwd})
    else:
        # Fallback: send 'y' or 'n' to the pane
        send_text(pane_id, "y" if resp != "reject" else "n")

def kill_agent(pane_id):
    subprocess.run(["tmux", "kill-window", "-t", pane_id])

def rename(pane_id, name):
    subprocess.run(["tmux", "rename-window", "-t", pane_id, name])
    subprocess.run(["tmux", "set-window-option", "-t", pane_id, "automatic-rename", "off"])
```

## SSE relay

The bridge does NOT poll. It subscribes once to monitor's SSE stream:

```python
async def monitor_sse_loop():
    async with httpx.AsyncClient(timeout=None) as client:
        async with client.stream("GET", "http://127.0.0.1:8769/api/stream") as r:
            async for line in r.aiter_lines():
                if line.startswith("data: "):
                    pane_state = json.loads(line[6:])
                    sessions = build_sessions(pane_state)
                    await broadcast_to_phones({"type": "status", "sessions": sessions})
```

Every WS-connected phone gets every update.

When the SSE goes down (monitor restart), the loop reconnects with backoff.

## Auth scopes (future)

For multi-phone scenarios, `scope` array in the token record:

| Scope | Allows |
|---|---|
| `status` | Read-only — pane state, output tail |
| `send` | Above + `send_text`, `send_key`, `permission_response` |
| `spawn` | Above + `spawn_agent`, `spawn_session`, rename |
| `admin` | Above + `kill_agent`, `kill_session` |

v0.6 phone always asks for `admin`. v0.7 may negotiate scope.

## Versioning

```json
{ "type": "auth", "token": "...", "client": "gmux-phone", "version": "0.6.1", "protocol": 1 }
```

Bridge replies in welcome:
```json
{ "type": "welcome", "protocol": 1, "server": "gmux-system 3.8.0", ... }
```

If `protocol` doesn't match, bridge closes with code 4400 ("bad protocol")
and a `{ "type":"error","message":"protocol mismatch: server=2, client=1" }`.

## How the phone reaches the host (transport)

| Transport | When to use | URL form |
|---|---|---|
| **LAN** | Same WiFi | `ws://192.168.1.20:8767` |
| **Tailscale** | Different network, both on tailnet | `ws://100.73.134.20:8767` |
| **Cloud relay** | No tailnet, behind CGNAT | `wss://relay.gmux.io/<token>` |
| **Self-hosted relay** | Privacy-conscious | `wss://your-vps.example/<token>` |

Bridge has no opinion — it just listens on `0.0.0.0:8767`. The phone is
told which URL to use via the pairing QR.

For Tailscale: `gmux pair` detects `tailscale ip -4` and uses that IP.

## Implementation order (v3.8)

1. `backend/bridge/auth.py` + `gmux pair` CLI (smallest surface, easiest test)
2. `backend/bridge/tmux_ops.py` + unit tests (no network, just tmux subprocess)
3. `backend/bridge/adapter.py` + unit tests (translate fixture pane state → session JSON)
4. `backend/bridge/bridge.py` skeleton: WS + HTTP server, handler stubs
5. Wire SSE subscription → broadcast to clients
6. Wire phone commands → tmux_ops
7. End-to-end test: spawn phone in mock mode → switch to live → confirm
8. systemd user unit for auto-start
9. Tauri sidecar registration (so launching the desktop app spawns the bridge)

Estimated: ~2-3 focused days. The phone is fully ready — every field
the spec needs is already in `monitor.py`'s output.

## What this enables

| Use case | How |
|---|---|
| Approve permission from phone while away from desk | Phone → `permission_response` → bridge → opencode POST |
| Spawn a new agent in a project from the bus | Phone → `spawn_agent` → bridge → tmux new-window + opencode |
| Watch a long-running agent without VPN | Tailscale or cloud relay |
| Voice-prompt an agent from the phone | Phone WS to `:8770` (voice daemon, already exists) → STT result → phone sends `send_text` |
| Send a redirection like "stop, do this instead" | Phone → `send_text` |
| Kill a runaway agent | Phone → `kill_agent` |
